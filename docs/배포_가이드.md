# Streamlit Cloud 배포 가이드 (상세)

이 문서는 GitHub와 Streamlit 둘 다 처음인 분도 따라할 수 있도록 단계별로 설명합니다.

## 사전 준비

- GitHub 계정 ([github.com](https://github.com) 가입, 무료)
- Git 설치 ([git-scm.com/download/win](https://git-scm.com/download/win))
- Python 3.9 이상

## 1단계: 로컬 환경 셋업

PowerShell을 열고 프로젝트 폴더로 이동:

```powershell
cd "D:\Prize_GA\주간 시책 검색 자동화"
```

가상환경 생성:

```powershell
python -m venv venv
venv\Scripts\activate
```

(활성화 성공 시 프롬프트 앞에 `(venv)` 가 보입니다.)

패키지 설치:

```powershell
pip install -r requirements-local.txt
```

설치 확인:

```powershell
python -c "import easyocr, rapidfuzz; print('OK')"
```

## 2단계: 정리 작업 (선택, 기존 데이터)

기존에 영문 파일명으로 저장된 8개 주차 (2604_1 ~ 2605_3)는 이미 매칭 정확도가 100%이므로 한 번에 처리할 수 있습니다.

```powershell
python scripts\match.py 2604_1 --no-ocr --no-interactive
python scripts\match.py 2604_2 --no-ocr --no-interactive
python scripts\match.py 2604_3 --no-ocr --no-interactive
python scripts\match.py 2604_4 --no-ocr --no-interactive
python scripts\match.py 2604_5 --no-ocr --no-interactive
python scripts\match.py 2605_1 --no-ocr --no-interactive
python scripts\match.py 2605_2 --no-ocr --no-interactive
python scripts\match.py 2605_3 --no-ocr --no-interactive
```

또는 한 줄로:

```powershell
foreach ($w in '2604_1','2604_2','2604_3','2604_4','2604_5','2605_1','2605_2','2605_3') {
    python scripts\match.py $w --no-ocr --no-interactive
```

결과는 `organized/` 폴더에 저장됩니다.

## 3단계: 로컬에서 Streamlit 앱 테스트

```powershell
pip install streamlit
streamlit run streamlit_app\app.py
```

브라우저가 자동으로 열립니다 (보통 http://localhost:8501). 월/주차/대리점을 바꿔보며 정상 작동을 확인하세요.

확인 후 Ctrl+C 로 종료.

## 4단계: GitHub 저장소 만들기

1. [github.com](https://github.com) 로그인
2. 우상단 `+` → `New repository`
3. 입력:
   - Repository name: 예) `weekly-policy`
   - **Public** 선택 ← 중요 (무료 Streamlit Cloud는 public만 가능)
   - "Add a README file" **체크 해제** (이미 있음)
   - .gitignore, license는 None
4. `Create repository` 클릭
5. 다음 화면에 나온 URL 복사 (예: `https://github.com/yunsangbai/weekly-policy.git`)

## 5단계: 로컬에서 GitHub로 첫 푸시

```powershell
cd "D:\Prize_GA\주간 시책 검색 자동화"

# Git 초기화 (처음 한 번만)
git init
git branch -M main

# 본인 정보 등록 (처음 한 번만, 본인 이메일/이름)
git config user.name "yunsangbai"
git config user.email "yunsangbai@gmail.com"

# 파일 추가
git add .gitignore README.md requirements.txt requirements-local.txt
git add data/ scripts/ streamlit_app/ docs/
git add organized/

# 첫 커밋
git commit -m "Initial commit: 시책 조회 시스템"

# 원격 저장소 연결 (위에서 복사한 URL)
git remote add origin https://github.com/YOURNAME/weekly-policy.git

# 푸시
git push -u origin main
```

푸시 시 GitHub 로그인이 필요합니다. 비밀번호 대신 **Personal Access Token**을 사용해야 합니다:

- GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
- "Generate new token" → `repo` 권한 체크 → 발급된 토큰을 복사
- 푸시 시 Password 자리에 이 토큰 붙여넣기

## 6단계: Streamlit Cloud 배포

1. [share.streamlit.io](https://share.streamlit.io) 접속
2. "Continue with GitHub"으로 로그인
3. "New app" 또는 "Create app" 클릭
4. 입력:
   - Repository: `YOURNAME/weekly-policy`
   - Branch: `main`
   - **Main file path: `streamlit_app/app.py`** ← 중요
   - App URL: 원하는 부분만 변경 가능
5. "Deploy" 클릭
6. 1-3분 빌드 후 앱 URL 표시됨
7. 그 URL을 북마크 / 공유하면 끝

## 문제 해결

### "git: command not found"
Git 설치가 안 되어 있어요. [git-scm.com/download/win](https://git-scm.com/download/win) 에서 설치.

### "Authentication failed" (푸시 시)
Personal Access Token이 필요합니다. 5단계 참고.

### Streamlit Cloud에서 한글이 깨져 보임
드문 경우인데, 폰트 문제일 수 있어요. `streamlit_app/.streamlit/config.toml` 파일을 만들어서 폰트를 지정할 수 있습니다 (필요시 별도 안내).

### "ModuleNotFoundError: No module named 'streamlit'" on Cloud
`requirements.txt`가 저장소 루트에 있고, 거기에 streamlit이 들어 있는지 확인.

### 푸시 용량이 크다고 거절됨 (>100MB)
이미지 파일 하나가 100MB를 넘을 수 없습니다. `.gitignore`로 원본 폴더는 제외되지만, organized/ 폴더의 개별 이미지가 크다면 미리 압축이 필요합니다.
